2019 年终总结与展望

Summary
  • 2019

时间飞逝，岁月如梭，转眼间 2019 年已经是最后一天了。这一年我还留在这家公司， 经历了很多，是真的多。回过头去看，不仅在技术上有了更多的技能树，对行业、对大环境也有了更深的理解，也有了更多的无奈，那就不吐不快吧。

业务相关

项目相关

  1. 一款金融业务 App 的数据驱动 -> 壳中壳 -> iOS 手动下架

    年初开始，制定了要精细化运作，要以数据驱动业务，嗯，那就先来 50 个 AB 试验当 OKR 吧。精益创业、增长黑客系列成为了当时的热点，AARRR模型、北极星指标、关键路径成为了重要知识点。马上就如火如荼地开展起来，业务和产品同学都懵了，你们怎么突然搞起AB来了…嗯是的，不强行 AB 就完不成 OKR 了，事实证明强行 AB 还是有一定效果的。首先从简单的UI上入手，先换文案、换颜色，再换图片、换动图，最后试验出来几个效果还不错的方案：挽留弹框（纯文案、数字吸引、图片、优惠券等手段递进）、缩短用户操作路径（能不要用户确认的就直接下一步，如自动填入可用金额、自动勾选等）、个人中心流量再利用（配置增收入口等）等等。整个过程持续到年中，还是很有意思的，而且对业务也有一定的反哺。

    同时，金融业务对动态化的依赖不言而喻，而这也同样意味着可替代。App 是 weex 的壳（容器），而现在的 weex 项目也可以引向另一个统一的 weex 项目。当确定停止业务迭代时，继续保有当前业务品牌且大家都换成同一个内核可能是对用户最低理解成本、对公司最低人力投入的方案了吧。当时觉得就这样把项目送出去了还有点不舍，现在看来也没什么，不过是一个壳中壳（对自己造的词还挺满意，真贴切）而已，毕竟现在连 iOS App 这个壳都被我手动下架了呵呵。

  2. 另一款金融业务 App 的审核与发版

    都说 iOS 要在面对全世界之前，先要过苹果爸爸这一关，在金融 App 上尤为明显。突如其来的 3.2.1 下架与无休止的 5.1.5 审核被拒，让每次的成功上架都显得弥足珍贵。表面上虽然是苹果爸爸的拦截，其实爸爸也是政策导向的，比如 315 的贷超问题和最近的 App 违规获取个人信息整改等。不过，这也给积极跟进政策整改的 App 一个成功上架的机会，很明显，我抓住了（这个牛皮可以吹不少时间呵）。

公司相关

大环境、互金行业、公司层面三者都遭遇了前所未有的困难，P2P 行业已经废了，大家都在积极转型，俗话说的跪下给银行、金融机构等大佬服务。公司遭遇 JC 叔叔上门后，陷入了很诡异的氛围，人人自危，担心自己的同事的同时，也担心自己也会进去；等事态缓和一些（陆续有小伙伴回归）后，还是开启了 CY 大戏，选择自告奋勇的同事更多是看空公司的未来（暴雷？）、看空年终奖，离开的都不难过（释怀？甚至有点开心？），而继续苟活着的人（比如我），也在战战兢兢地想着会不会被抓走、年终有没有着落等等，所以让自己忙起来可能就不会有闲功夫去胡思乱想些无力的事情了。

个人成长

技术相关

  1. 前端 weex

    一朝天子一朝臣，某天起 weex 不再是公司支持使用的技术栈，嗯，好的。那我重新学习一下 H5 吧，求推荐好用易上手的前端脚手架，就是那种可以拿来直接做项目的那种。

  2. 后端 Java

    基于 Sping+SpingMVC+Mybatis+Shiro+Redis+MySQL 实现了一个类似于购物的项目，希望这个项目后面可以起来吧。这一套东西相比于十年前的 Spring+Struts2+Hibernate 的确好用不少，当然还没究极进化到使用 SpringBoot，idea IDE也比 Eclipse/MyEclipse 好用一万倍，另外插件也能提升很多工作效率，再次希望自己能一直这样学习运用下去，保持好奇心。也感谢在这方面帮助过我的各位服务端大佬。

  3. 微信小程序

    之前已经上架的微信小程序《尬聊终结神器》，是一款可以自定义生成表情包的小程序。十一前突然收到推送，说我的小程序涉嫌未设置过滤违法、违规等不当信息内容的机制行为，已被下架处理；本着自救自查的态度，发现问题就是出在“自定义生成”上了，用户可以自行输入想表达的内容而没有接入过滤功能，这个时间点也是可以理解的，屏蔽自定义入口后重新提交审核，通过审核重新上架。

    今年还上架了一款《看看你的二次元形象》小程序，主要是将自拍照片转化成二次元头像的；基于 U-GAT-IT 论文实现的图片转换功能，当然算法不是我写的，我只是调用已经训练好的模型生成一下，不过在过程中也用 Tensorflow 跑过 demo，了解了一下原理。

  4. 爬虫相关

    今年国家政策对爬虫的打压上升到了法律层面，只允许对完全公开的数据进行爬虫存储，否则就以破坏计算机系统来论处。

    saving是一款帮你在资料网省钱的命令行下载工具，目前只支持部分文件格式的下载。

  5. 自动化相关

    WeChatMediaPlatformAutomation是一款在微信公众号自动发布文章的命令行工具。

    wechat-format-cli是一款将 Markdown 文本转化成可直接用于微信公众号编辑器的命令行工具。

2019 规划总结

  1. 技术书籍（偏前端）5 本，《Web前端开发精品课》（一点都不精品，没两下就看完了，都是非常基础的知识）、《前端程序员面试笔试宝典》（用于查漏补缺挺好的，也算比较基础的知识）、《高效前端：Web 高效变成与优化实践》（实践出真知，很不错，还没有全部读完）、《微信小程序策划与运营》（微信官方文档的白话文+demo，外加一些成功案例，给新手当科普还行吧）、《Tensorflow 第二版》（非常注重实践的书，目前没有打算深入，只能后续当工具书来用了），严格来说是没有完成的，偷个懒勉强算完成吧。

  2. 软实力相关书籍 5 本，实际上最后理解为非技术书籍可能会更准确一点，刘润《新零售》（很不错的书，将新零售说的很透，推荐）、《让大象飞》（创业方面的书，有些观点很新奇：不要错失生活本身等，还没完全读完）。

  3. 带娃出国旅游 1 次，双十一带娃去了日本的大阪、京都奈良转转，等娃长大点出去玩应该会方便不少。

2020 新的规划

  1. 技术书籍 5 本

  2. 非技术书籍 3 本

  3. 考上教师、系统架构设计师

  4. 带娃国内旅游 2 次

2020，come on！
